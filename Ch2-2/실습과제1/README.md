## 라이브러리는 헤더파일과 목적파일로 구성되어 있다. 
```헤더파일과목적파일에 들어 있는 내용이 무엇인지 각각 설명하시오.```

- 헤더파일은 소스에서 함수/클래스를 선언 할 때 사용합니다.
- 목적파일은 실제로 함수/클래스가 구현 되어있는 파일입니다.

---

## 함수의 선언, 정의, 호출은 각각 어디에 존재하는가?

- 함수 선언: 헤더파일에 위치
- 함수 정의: 소스 파일에 위치
- 함수 호출: 함수를 실행하는 코드에서 사용
---

## <iostream> 헤더파일에 cin, cout은 extern 변수로 선언되어 있다.
```extern 선언과 변수선언의 차이를 설명하라. 그리고, 그럼 cin, cout의 변수선언은 어디에 있는가?```
- 변수 선언 : 실제로 메모리를 할당하고 변수 생성한다.
- extern 변수 : 이미 존재하는 변수를 다른 파일/위치에서 사용한다고 알려주는 것이다.

- `<iostream>` 헤더 파일 안에서는 `cin, cout`이 `extern`으로 선언되어 있음

---

라이브러리 함수나 객체를 사용하기 전에 헤더파일을 include하는이유를 설명하라.

- 라이브러리 함수나 객체가 존재한다는 것을 컴파일러에게 알려주고, 올바르게 사용할 수 있도록 정보 제공하는 역할입니다.

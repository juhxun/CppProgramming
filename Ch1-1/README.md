## 강의노트에서 몰랐던 용어에 대하여 자세히 조사하시오.
### STL (Standard Template Library) 라이브러리

### 라이브러리란?
- 프로그램에서 사용할 수 있도록 미리 만들어져 있는 함수와 변수들의 모음집이다.
### STL이란?
- C++에서 제공하는 표준 라이브러리로, 자주 사용하는 **자료구조와 알고리즘을 모아 둔 것**이다.
STL은 4가지로 구성된다.

### 1️⃣ Container (컨테이너)
- 데이터를 저장하는 자료구조  

### 2️⃣ Iterator (반복자)
- 컨테이너의 요소를 가리키는 객체 (포인터와 비슷한 역할)  

### 3️⃣ Algorithm (알고리즘)
- 정렬, 탐색 등의 기능 제공  

### 4️⃣ Function Object (함수자)
- 함수처럼 동작하는 객체 (`operator()` 오버로딩)
---
## 알고리즘 설계와 코딩 중 무엇이 중요하고 그 이유는 무엇인가?

**알고리즘 설계가 더 중요한것 같다.**

교수 님이 말씀하셧듯 알고리즘은 문제를 해결하는 방법이고, 코딩은 구현하는 과정으로 반복작업이다.  

---

## 클래스와 객체란?

- **클래스(Class)** : 사용자 정의 자료형
- **객체(Object)** : 클래스형 변수

---

## 절차지향 vs 객체지향 프로그래밍

| 구분 | 절차지향 | 객체지향 |
|------|-----------|-----------|
| 중심 | 함수(절차) 중심 | 객체 중심 |
| 특징 | 위에서 아래로 순차 실행 | 데이터와 기능을 하나로 묶음 |
| 예 | C | C++, Java |

👉 객체지향은 재사용성과 유지보수가 좋다.

---

## 라이브러리란?

라이브러리는 **미리 만들어 둔 함수와 변수들의 모음**이다.  

---

## 라이브러리가 목적파일 형태로 존재하는 이유

라이브러리는 보통 **목적파일(Object File)** 형태로 제공된다.

이유:
- 소스코드를 공개하지 않아도 된다.
- 컴파일 시간을 줄일 수 있다.
- 재사용이 쉽다.

즉, 이미 기계어로 번역된 상태로 제공되어  
바로 연결만 하면 사용할 수 있다.

---

## 링크(Link) 과정이 필요한 이유

프로그램은 여러 파일로 나뉘어 작성된다.

링크 과정은:
- 여러 목적파일을 하나로 합치고
- 필요한 라이브러리를 연결하여
- 최종 실행파일(.exe)을 만드는 과정이다.

👉 함수의 실제 위치를 연결해 주기 위해 반드시 필요하다.

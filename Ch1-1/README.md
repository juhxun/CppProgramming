## 강의노트에서 몰랐던 용어에 대하여 자세히 조사하시오.
### STL (Standard Template Library) 라이브러리

### 라이브러리란?
- 프로그램에서 사용할 수 있도록 미리 만들어져 있는 함수와 변수들의 모음집이다.
### STL이란?
- C++에서 제공하는 표준 라이브러리로, 자주 사용하는 **자료구조와 알고리즘을 모아 둔 것**이다.
STL은 4가지로 구성된다.

### 1️⃣ Container (컨테이너)
- 데이터를 저장하는 자료구조  

### 2️⃣ Iterator (반복자)
- 컨테이너의 요소를 가리키는 객체 (포인터와 비슷한 역할)  

### 3️⃣ Algorithm (알고리즘)
- 정렬, 탐색 등의 기능 제공  

### 4️⃣ Function Object (함수자)
- 함수처럼 동작하는 객체 (`operator()` 오버로딩)

## 알고리즘 설계와 코딩 중 무엇이 중요하고 그 이유는 무엇인가?

**알고리즘 설계가 더 중요한것 같다.**

교수 님이 말씀하셧듯 알고리즘은 문제를 해결하는 방법이고, 코딩은 구현하는 과정으로 반복작업이다.  

---

## 클래스와 객체가 무엇인지 설명하시오.

- **클래스** : 사용자 정의 자료형
- **객체** : 클래스형 변수

---

## 절차지향 프로그래밍 언어와 객체지향 프로그래밍 언어의 차이를 조사하라.

 절차지향 프로그래밍
- **함수(절차) 중심**으로 프로그램을 구성한다.
- 데이터와 함수가 분리되어 있다.

특징
- 작은 규모의 프로그램에 적합하다.
- 코드 재사용성과 유지보수가 상대적으로 어렵다.

대표 언어
- C 언어

객체지향 프로그래밍 (Object-Oriented Programming)
- 객체 중심으로 프로그램을 구성한다.
- 데이터와 기능(함수)을 하나로 묶어 관리한다.
- 클래스(Class)를 기반으로 객체를 생성한다.

특징
- 재사용성이 높다.
- 유지보수가 쉽다.
- 대규모 프로그램에 적합하다.

대표 언어
- C++
- Java
- Python

**절차지향은 “함수 중심”, 객체지향은 “객체 중심” 프로그래밍 방식이다.**
---

## 라이브러리가 무엇인지 자세히 설명하시오.

- 프로그램에서 사용할 수 있도록 미리 만들어 둔 함수, 클래스, 변수 등의 모음집이다.  
- 개발자는 자주 사용하는 기능을 직접 구현하지 않고, 라이브러리를 불러와서 사용함으로써 개발 시간을 줄일 수 있다.
---

## 라이브러리가 목적파일 형태로 존재하는 이유를 설명하라.
이미 기계어로 번역된 상태로 제공되어  바로 연결만 하면 사용할 수 있다.

- 컴파일 시간을 줄일 수 있다.
- 재사용이 쉽다.

---

## 링크과정이 필요한 이유를 설명하라.
프로그램은 여러 파일로 나뉘어 작성된다.

- 여러 목적파일을 하나로 합치고, 최종 실행파일(.exe)을 만드는 과정이다.

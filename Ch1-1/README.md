# 알고리즘 설계와 코딩 중 무엇이 더 중요한가?

## 1. 결론

👉 **알고리즘 설계가 더 중요하다.**

코딩은 설계된 알고리즘을 구현하는 과정이지만,  
알고리즘이 잘못 설계되면 아무리 코딩을 잘해도 좋은 프로그램이 될 수 없다.

---

## 2. 알고리즘 설계가 중요한 이유

### 1️⃣ 문제 해결의 핵심 단계이기 때문이다
알고리즘은 **문제를 어떤 방식으로 해결할지에 대한 논리적 절차**이다.  
설계가 잘못되면 프로그램은 올바른 결과를 낼 수 없다.

예:
- 비효율적인 알고리즘 → 실행 속도가 매우 느려짐
- 잘못된 로직 → 잘못된 결과 출력

---

### 2️⃣ 성능을 결정한다
프로그램의 실행 속도와 메모리 사용량은  
대부분 **알고리즘의 시간 복잡도(Time Complexity)** 와  
**공간 복잡도(Space Complexity)** 에 의해 결정된다.

예:
- O(n²) 알고리즘 vs O(n log n) 알고리즘  
→ 데이터가 많아질수록 성능 차이가 매우 커짐

---

### 3️⃣ 코딩은 도구, 알고리즘은 설계도이다
건물을 지을 때 설계도가 잘못되면  
아무리 시공을 잘해도 좋은 건물이 될 수 없다.

- 알고리즘 = 설계도
- 코딩 = 시공

즉, **설계가 우선이고 구현은 그 다음 단계이다.**

---

## 3. 그렇다면 코딩은 중요하지 않은가?

❌ 그렇지 않다.

코딩 또한 중요하다.

- 문법 오류 없이 정확히 구현해야 한다.
- 가독성 좋은 코드 작성이 필요하다.
- 유지보수를 고려해야 한다.

하지만 **좋은 알고리즘 없이 좋은 코드는 존재하기 어렵다.**

---

## 4. 정리

| 구분 | 알고리즘 설계 | 코딩 |
|------|----------------|-------|
| 역할 | 문제 해결 방법 설계 | 설계를 실제 코드로 구현 |
| 영향 | 정확성, 성능 결정 | 완성도와 유지보수성 결정 |
| 중요도 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

📌 **결론적으로, 알고리즘 설계가 더 중요하지만, 두 요소는 서로 보완 관계에 있다.**  
좋은 개발자는 좋은 알고리즘을 설계하고, 그것을 정확하고 깔끔하게 구현할 수 있어야 한다.

# 실습과제1
## 접근지정자가 왜 필요한가?
- 코드의 캡슐화 및 정보 은닉, 유지보수 등 여러 이유가 있다.
> 클래스 내부를 아무나 마음대로 접근하면 문제가 생길 수 있다. 그래서 private, public 등으로 접근 범위를 제한한다. 그리고 내부 코드를 바꿔도 외부에서 사용하는 방식으로 유지보수가 쉽다.
##  생성자와 setter 멤버함수는 모두 멤버변수를 설정하는 기능이다. 차이점을 설명하라.
| 구분    | 생성자     | Setter  |
| ----- | ------- | ------- |
| 호출 시점 | 객체 생성 시 | 객체 생성 후 |
| 호출 횟수 | 1번      | 여러 번 가능 |
| 목적    | 초기화     | 값 변경    |
| 호출 방식 | 자동      | 직접 호출   |
#  객체지향형 언어의 특징인 캡슐화와 정보은닉을 설명하라.
- 캡슐화는 데이터와 기능을 하나로 묶는 것이며, 정보은닉은 접근지정자를 사용하여 객체 내부의 데이터를 외부로부터 보호하는 것이다.
